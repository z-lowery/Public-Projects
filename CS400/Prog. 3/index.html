LAMBDA EXPRESSIONS

What are Lambda Expressions?

...and how do they work?

Abstract Classes vs. Lambda Expressions

  • Abstract Classes
    • A programmer utilizes an Abstract Class when they instantiate a new object and define it there. For example:

      	myClass className = new myClass(){
      		methodName(){
      			someCode;
      		}
      	}

    • Concisely describes implementation for a single object
    • Convienient for simple and single/limited use objects
    • Cannot reference type (no name) making testing difficult
  • Lamba Expressions
    • More concise syntax for objects with a single abstract/undefined method
    • You can not use a lambda expression if the object or interface doesn't have just one abstract/undefined method

Using a Lamba Expression

  • To use a lamba expression, the programmer must first create either a) A functional interface or b) A class with ONE abstract method.

    This is the method that you will be creating a lambda expression for.
  • A programmer can utilize a lamba expression in different ways. One way is writting the method definition directly:

    	myObject object = (x,y) -> {
    		someCode;
    	}

    OR, my just providing what the method should return:

    	myObject object = (Integer x, Integer y) -> x + y;
     
    	  Notice how no "return" statement is used to return  x + y
  • (x, y) = the parameter(s) (seperated by a comma ',') utilized in the abstract method. If the method only utilizes one parameter, you do not need the brackets. If it utilizes no parameters, use "()".
  • Any calls to "object" within the class will utilize the provided class definition
  • Because of this, you can create multiple definitions to call in the class! For example, you can create another object, say, object2, with a different method definition
